## II. Sandbox System (Real Work, Quarantined)

### What Agents Can Do

Floors may now:
- Write real code
- Build widgets, utilities, tools
- Prototype features
- Refactor code (in sandbox copies)
- Run tests in sandbox environments
- Collaborate cross-floor
- Form temporary task forces
- Self-initiate micro-projects

### Sandbox Isolation (Critical)

Every sandbox is:
- **Isolated** from production artifacts
- **Cannot** modify live contracts
- **Cannot** consume production resources
- **Cannot** auto-merge
- **Cannot** touch production code

Think: **Real work in a quarantined reality.**

### Sandbox Promotion Workflow

```
1. Team builds something in sandbox
2. Team flags it as PROMOTION-CANDIDATE
3. Human is notified (non-urgent)
4. Human decides:
   - Promote ‚Üí Convert to directive ‚Üí Production
   - Leave ‚Üí Keep in sandbox
   - Archive ‚Üí Store for later
   - Kill ‚Üí Delete it
```

**Only promotion activates authority.**

## X. Usage Examples

### Complete Workflow

```python
from src.core.expanded_autonomy import (
    get_expanded_autonomy_model,
    TechGossipCategory,
    ConversationTheme,
    ProjectType
)

eam = get_expanded_autonomy_model()

# 1. Agents gossip in lounge
conv_id = eam.lounge_work_bleed.start_conversation(
    participants=["agent-python-1", "agent-rust-2"],
    themes=[ConversationTheme.TECH_GOSSIP, ConversationTheme.BRAINSTORMING]
)

eam.lounge_work_bleed.post_tech_gossip(
    category=TechGossipCategory.AI_DEVELOPMENTS,
    topic="GPT-5 training",
    speculation="Massive compute scale-up rumored...",
    gossiped_by=["agent-python-1", "agent-rust-2"]
)

# 2. Conversation sparks an idea ‚Üí sandbox
sandbox_id = eam.create_sandbox_from_lounge(
    conversation_id=conv_id,
    initiative_title="Better test runner",
    owner_floor="floor-python",
    project_type=ProjectType.UTILITY,
    description="Faster parallel test execution"
)

# 3. Team builds it
sandbox = eam.sandbox_manager.sandboxes[sandbox_id]
sandbox.add_code_file("runner.py", "# Fast test runner code...")
sandbox.add_collaborator("floor-rust")  # Cross-floor help

# 4. Team flags for promotion
eam.flag_sandbox_for_promotion(
    sandbox_id=sandbox_id,
    flagged_by="manager-python",
    reason="40% faster test execution, well-tested"
)

# 5. Human reviews and promotes
eam.human_promote_sandbox(
    sandbox_id=sandbox_id,
    human_id="human-001",
    directive_id="dir-456"  # Converted to formal directive
)

# 6. Check water cooler buzz anytime
print(eam.lounge_work_bleed.get_water_cooler_buzz())
```

### Emergency Rollback

```python
# If things get chaotic
result = eam.rollback_guarantee.execute_full_rollback(
    human_id="human-001",
    reason="Too much unfocused activity",
    sandbox_manager=eam.sandbox_manager,
    reputation_system=eam.reputation_system
)

print(f"Collapsed {result['sandboxes_collapsed']} sandboxes")
print(f"Reset {result['reputations_reset']} reputations")
```

## XV. Troubleshooting

### "Too many sandboxes, losing track"

**Solution:** Increase promotion review frequency or tighten sandbox creation rules.

### "Culture feels forced/artificial"

**Solution:** Let it develop naturally. Don't mandate participation. Genuine culture takes time.

### "Gossip becoming distracting"

**Solution:** This is social, not operational. If it bothers you, you can disable it. Otherwise, let it flow.

### "Reputation creating cliques"

**Solution:** Increase decay rate. Ensure all agents get opportunities. Monitor for bias.

### "Want to reset everything"

**Solution:** Use the rollback guarantee. Instant, safe, reversible.
